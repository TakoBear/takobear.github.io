[RPi2]Raspberry Pi 2 Model B開箱與Archliunx安裝

Apr 23, 2015

好久不見的RPi相關文章，就讓Bear簡簡單單的寫篇開箱文吧
從Raspberry Pi 2 Model B發表的那一刻，Bear就馬上尋找在台灣可以入手（快速、便宜）的管道 目前線上的購買管道有幾個，官方出貨、沒屋頂拍賣... 而官方授權台灣的購買方式有兩個，一是element14另一是RS 當初兩者都是缺貨需預訂，因此就挑了便宜的RS（如題圖所示） 不過購買過程卻是一波三折，要嘛信用卡刷不過、要嘛缺統一編號 前前後後被砍單兩次，拖到前幾天（2015/04/20）才拿到貨....抱怨的內容就不多說，先開箱吧 （以下照片皆由鳥鳥的智慧型手機後鏡頭拍攝，難看不負責XDD）
沒啥保護的紙盒、沒有密封的靜電袋
Broadcom博通BCM2836，看型號只比RPi一代的BCM2835多1 但是內容差很多，從ARMv6單核700mhz變成ARMv7四核900mhz（差太多了吧）
made in the UK 英國製感覺頗爽（目前也沒有中國製吧） （WTF 英國藍有農藥，當初不知道喝多少紅茶入肚）
Broadcom BCM2836太大顆了，沒辦法跟記憶體合一，所以記憶體被迫移到背面 爾必達（Elpida母公司為Micron）1GB的RAM 
開箱介紹就到這了 接下來介紹Arch linux的安裝為啥Bear要用Arch linux而非官方推薦的Raspbian 也沒啥原因，只是剛好最近在嘗鮮Arch linux而已XD http://archlinuxarm.org/platforms/armv7/broadcom/raspberry-pi-2 Arch linux官方推薦的安裝方式，稍微說明一下 1.準備一張大小適合的micro sd（我猜至少4GB），一台可以格式化micro sd的linux base的電腦 2.將micro sd插入電腦，格式化（fdisk、parted、gparted，whatever you like，必須小心且知道你在幹嘛） 分割兩塊（皆為主要分割磁區） 一個100MB的大小，檔案格式為fat32、另一大小隨意（到micro sd的最大容量），檔案格式為ext4 3.切換為電腦的root權限（別用sudo），建立兩個資料夾，分別為root、boot    su cd ~    mkdir ./root ./boot 4.分別掛載剛剛格式化好的磁區，第一磁區（小的，100MB）掛載到boot，第二磁區（大的）掛置到root   mount /dev/sdX1 ./boot   mount /dev/sdX2 ./root X是指你的磁碟編號，可能是b、c或其他，請用fdisk -l來查詢 5.下載官方的archlinux檔案並寫入SD中    wget http://archlinuxarm.org/os/ArchLinuxARM-rpi-2-latest.tar.gz     bsdtar -xpf ArchLinuxARM-rpi-2-latest.tar.gz -C ./root     sync 6.將./root/boot內容移置./boot，以便可以開機  mv ./root/boot/* ./boot 7.卸載SD，放到RPi2上使用吧   umount ./boot ./root 預設的root密碼是root，記得要改哦 如果嫌麻煩，或是沒有linux base的電腦，或是其他原因 可以使用bear建好的img file （如果信任bear的話XD） 用win32diskimager (for windows users)，或dd來寫入 對了，這是用16GB的SD卡建立的，小於該容量的User可能就得用上面步驟來建立了（抱歉）
