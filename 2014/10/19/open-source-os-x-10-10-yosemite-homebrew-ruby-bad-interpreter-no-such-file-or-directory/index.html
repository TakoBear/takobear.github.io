[Open Source] 升級成OS X 10.10 (Yosemite) - Homebrew 發生 ruby bad interpreter: No such file or directory

Oct 19, 2014

最近應該蠻多朋友興高彩烈的升級成OS X 10.10 (Yosemite)吧, 不過如果你是喜歡用終端機(Terminal)進行編譯開發的朋友, 可能會發現最近你愛用的brew 命令不能使用了... Bear也是今天測試的時候才發現, 但是要解決它並不困難喔!
今天原本Bear要跟之前一樣安裝新套件, 不過沒想到一下brew install 的指令就出現以下錯誤.../usr/local/bin/brew: /usr/local/Library/brew.rb: /System/Library/Frameworks/Ruby.framework/Versions/1.8/usr/bin/ruby: bad interpreter: No such file or directory 這種事情真的麻煩, 每次作業系統升級時可能或多或少都會遇到, 要解決這個問題的話步驟蠻簡單的! 這邊主要是brew的指令沒有正確對應到路徑, 只要稍微調整一下就好了!! 步驟如下: 1) 打開你的終端機 （Terminal） 2) 輸入 nano /usr/local/Library/brew.rb 3) 把你看到的畫面第一行有一個 "1.8" 改成 "Current" 就可以解決啦!! #!/System/Library/Frameworks/Ruby.framework/Versions/Current/usr/bin/ruby -W0
改完後按下ctrl + x就搞定啦!! 換版本這陣子應該陸陸續續還會有很多問題, TakoBear如果發現有更多問題會再通知大家如何解決!!
