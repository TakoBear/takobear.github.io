    建立 OS X Application 專案

    執行Xcode，點擊 Create a new Xcode project。

    選擇左側 OS X 下的 Application，再選擇右側畫面的 Cocoa Application，點擊 Next。

    這邊就要開始認識到, Mac的開發上, 著重的Framework就是從Cocoa開始的,

    在Mac上沒有UIKit的概念, 相對的著重在於AppKit, Cocoa以及Foundation等基礎的Framwork,

    接下來我們在慢慢往下介紹吧

    依序輸入如圖中的資訊。

    Product Name:就是你的 APP 名稱。

    Organization Name:組織名稱，可以不填。

    Company Identifier:這會套用在 Bundle Identifier。

    Bundle Identifier:用來識別 APP 的唯一識別串字串。

    Class Prefix:class 的前綴字串，用在 Application Delegate的名稱上：Class Prefix + AppDelegate(.h/.m)。

    App Store Category:App Store 的類別，本練習填 none 就可以了。

    Document Extension:本練習不填, Bear之後會在慢慢介紹

    選擇專案存放路徑，點擊 Create。

    就會產生如圖專案基本所需的設定與檔案。

    專案說明

    點擊 HelloWorld，中間畫面就會出現專案設定。

    HelloWorld下放置了最基本的程式檔。

    bearAppDelegate(.h/.m):預設的程式碼。

    MainMenu.xib:提供預設的選單與空視窗。

    Images.xcassets:專案Icon所參考的目錄

    Supporting Files:基本上不會直接動到裡面的檔案

    點擊執行。

    執行結果：選單與空視窗。

    設計版面(layout)

    接下來給 APP 提供一些互動功能。

    跟iOS不太一樣, 在開發行動裝置時, 我們很長需要計算到滿畫面也就是最大寬度或最大高度解析度下的畫面中,

    元件需要怎麼配置的問題, 但是在桌機的應用程式開發中, 其實對於版面的配置要求跟手機上的需求確實是不同的

    在手機上的開發, Bear不太建議大家採用這種XIB概念設計, 因為變動幅度實在太大而且機率太高了,

    也為了能夠快速開發每個不同的專案, 採用程式化的開發才比較容易減輕之後的負擔,

    當然這裡說多了, 其實Bear只是想簡化教學而已XD

    更換 APP 圖示(icon)

    這裡Bear得提一下, 如何在Mac App內更換掉原本的iCon方法, 遵照以下步驟才能把你的Mac App icon變得跟其他人不一樣噢!

    點開 Products ，點擊 HelloWorld.app 可以看到目前 APP 所使用的圖示。

    選擇 Images.xcassets、選擇 Appicon，就會顯示不同解析度的 icon 設定畫面。

    關於 icon 的解析度大小，請參考 OS X Human Interface Guidelines。

    製作不同解析度的 icon 圖片。

    將圖片依照解析度，拖曳至適當位置。如果螢幕寬度太小，無法看到 256 或是 512 的欄位，可以先關閉側邊欄。

    接著重新產生 APP

    按下鍵盤上的 Command + Shift + K，清除專案之前編譯的檔案。

    按下鍵盤上的 Command + R，編譯並執行程式。

    點開 Products ，點擊 HelloWorld.app 可以看到目前 APP 所使用的圖示已經換成凝視的熊臉了。

    到這邊是不是覺得很有趣呢XDDDD

    對 HelloWorld.app 點擊右鍵叫出選單，點擊 Show in Finder。

    點選Show In Finder, 就能發現你的App所在的資料夾位址囉

    有使用Mac的朋友們, 大家對這種狀況應該不陌生吧? 一款Mac App下載下來後, 最重要的就是把這個編譯出來的熊臉...

    呃不是, 是App放到應用程式的資料夾內, 一般只要點選他就能啟動程式了

    到這邊是不是理解整個結構了呢?
