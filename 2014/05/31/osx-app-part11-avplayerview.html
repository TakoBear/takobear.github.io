OSX APP 教學 Part11-AVPlayerView

May 31, 2014

本次教學目標：

  1. 如何使用 AVPlayerView 製作影片播放器

    2. 操作環境：

    OS X Version 10.9.3

    Xcode Version 5.1.1 (5B1008)

    在 OS X v10.9之前要播放影片，開發者必須要在版面上設計控制按鈕與呈現影片的動畫視圖。但在 OS X v10.95 之後，我們可以使用AVPlayerView ，Apple 提供的播放器界面視圖。AVPlayerView 擁有播放控制項與時間軸，只要提供 AVPlayer，就可以如同一般的影片播放器正常運作。

    本次教學會教各位如何使用 AVPlayerView 快速製作一個簡單的影片播放器。

    建立 OS X Application 專案

    這次我們建立一個名為 HelloAVPlayerView 的專案請參考教學 OSX APP 教學 Part2-Timer 建立新專案

    新專案名為 HelloAVPlayerView。

    切換至專案的 Build Phases 頁面。

    於 Link Binary With Libraries 加入AVFoundation.framework 與 AVKit.framework。

    AVFoundation.framework 與 AVKit.framework 是提供影像相關的開發框架。本次教學的程式需要引用到，AVPlayerView 與 AVPlayer 等類別都定義在這些開發框架中。

    新增 Group 名為 Resources, Controllers 與 Views。

    新增類別 MainWindowController 繼承至 NSWindowController。

    新增類別 DraggedAVPlayerView 繼承至 AVPlayerView。

    整理專案結構如下：

    設計版面(layout)

    刪除 Main.xib 的 Window。

    選擇 MainWindowController.xib 將 AV Player View 元件拖至 Window 上。

    接著為了能夠在調整 Window 尺寸時， AVPlayerView 的尺寸也會跟著變動，所以我們要加上 AutoLayer 的功能。

    選擇 Player View，再點選圖中的按鈕 Resolve Auto Layer Issues。

    點擊 Add Missing Constrains。

    Constrains 就是自動佈局用的限制條件，在調整 window 尺寸的時候，AVPlayerView 就會跟著 Window 的尺寸變動。

    接下來將開始編寫程式碼了。bearAppDelegate.h/.m 請參考 OSX APP 教學 OSX APP 教學 Part2-Timer

    選擇 MainWindowController.xib， 將 Player View 的 Custom class 屬性改成 DraggedAVPlayerView。

    對 File’s Owner 點擊右鍵，依圖連接元件。

    測試結果

    接下來測試一下完成的程式。

    按下鍵盤上的 Command + Shift + K，清除專案之前編譯的檔案。

    按下鍵盤上的 Command + R，編譯並執行程式。

    將影片檔拖曳至程式中，就會開始播放影片。

    後續討論

    如果想要利用程式操作影片控制，只要呼叫 player 的方法就可以了。不過本次教學的影片播放器的支援格式只有：

    QuickTime

    MPEG4(.mp4, .mp4, .m4v)

    3GPP

    Wave, AIFF, CAF

    MP3, AAC

    其他的格式如 mkv，就要另外想辦法處理了。
