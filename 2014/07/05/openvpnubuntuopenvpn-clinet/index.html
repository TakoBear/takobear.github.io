[OpenVpn]在Ubuntu下執行openvpn clinet

Jul 5, 2014

還記得利用RPI建立openvpn server以及Windows底下執行openvpn client嗎?
小編今天要介紹大家怎麼在Ubuntu透過openvpn翻牆囉!
無論是server端，還是client端，都必須安裝完整的openvpn才能正常運作
所以第一步

sudo apt-get update && sudo apt-get install openvpn

完成後建立client的文件檔(XXX.conf)，備妥所需要的憑證檔(紅字所述)
將上述文件放置在一起

client
dev tun
proto tcp
remote YourServerIP YourServerPort
resolv-retry infinite
nobind

persist-key
persist-tun
ca /絕對路徑/ca.crt
cert /絕對路徑/UserName.crt
key /絕對路徑/UserName.key

;auth-user-pass
ns-cert-type server

tls-auth tls.key 1
comp-lzo
verb 3

接下來就執行看看是否能運作

sudo openvpn /絕對路徑/XXX.conf

出現Initialization Sequence Completed的話就表示正常連線了:)

當然問題沒這麼簡單，會發現怎麼ping IP都可以ping的到
但是網頁瀏覽確有問題呢?
其實是因為vpn server的DNS無法順利提供給client端來使用
client端缺少DNS解析域名所以無法判別

暴力一點的方法就是直接修改client端的/etc/resolv.conf
增加所需要的DNS server IP(ex:168.95.1.1 or 8.8.8.8)

優雅一點就是使用openvpn提供的腳本檔來修改
會在使用vpn服務時，自動從server取得DNS
那就會稍微複雜一點

首先先安裝resolvconf來控制resolv.conf

sudo apt-get install resolvconf

刪除舊的resolv.conf，並使用resolvconf提供的resolv.conf作為代替
切記ln必須使用絕對路徑

sudo rm /etc/resolv.conf
sudo ln -s /run/resolvconf/resolv.conf /etc/resolv.conf

重啟resolvconf服務

sudo /etc/init.d/resolvconf restart

再來編輯剛剛的openvpn client的文件檔(XXX.conf)
增加下面三行

script-security 2
up /etc/openvpn/update-resolv-conf
down /etc/openvpn/update-resolv-conf

完成後，執行vpn就會自動取得DNS了
