Bear馬上又要來介紹如何利用iOS的AudioToolbox Framework來播放mp3 音樂囉!這次內容分成兩篇進行, 先介紹AudioToolBox如何使用

下一篇我們會介紹使用AudioToolbox 配合FFmpeg制作網路收音機

但基礎的部分還是要從這篇開始囉!

其實播放音樂或是影片的播放器(Player)概念都是這樣的流程取出檔案來源(可能是本地端, 或是遠端)

- >編碼 (encode)每一個buffer呈frame 送出去

-> 解碼 (decode)成一堆buffer

-> 播放

實際上與遠端運作會牽涉到一種名為RTSP(Real Time Streaming Protocol)類似的操作

因為範疇過大, 這篇我們不會深究這樣的Protocol運作方式

如果要對檔案進行編碼, 解碼的操作

我們需要瞭解不同檔案的格式

這邊我們先介紹最簡單的AAC格式是什麼

讓我們先抄一下Wiki的定義...

AAC (Advanced Audio Coding)

副檔名.m4a, .m4b, .m4p, .m4v, .m4r, .3gp, .mp4, .aac

網路媒體型式audio/aac, audio/aacp, audio/3gpp, audio/3gpp2, audio/mp4, audio/MP4A-LATM, audio/mpeg4-generic

其實大家只要知道這幾種副檔名都會被歸類為是AAC格式

而Apple iOS 設備上對於AAC格式的檔案播放是有限制的!

有支援的檔案格式可以直接用AVAudioPlayer或是AVPlayer兩種class來播放

甚至是使用AudioToolbox的方式直接進行播放

而如果是Apple不支援的檔案格式呢

我們就是用FFmpeg來把他解壓縮成PCM格式( 全名為: Pulse-Code Modulation, 為一種標準的音訊格式)

然後再用AudioToolbox的功能來播放!!

介紹這篇的原因在於

雖然AVPlayer或是AVAudioPlayer都已經讓播放音樂這件事情變得很簡單

但是當我們必須要支援不同的聲音格式的時候

這兩種Class變成能夠擴展的格局就不夠了

為此我們還是必須回來用古早的AudioToolbox來處理囉!

這邊Bear決定先不要再往下探討AAC格式如何播放以及FFmpeg的搭配...

我們先討論如何使用AudioToolbox 這個Framework來播放mp3音樂吧!!!

以AAC為例的播放音訊過程就可以參考下圖囉!

那麼~~廢話就不多說我們馬上開始這次的主題吧!

這次建立的Class實體需要有的Method配置如下

(記得要先到Build Setting內加入 AudioToolBox framework!)

#import <Foundation/Foundation.h>

#import <AudioToolbox/AudioToolbox.h>

#import <AudioToolbox/AudioFile.h>

@interface SHAudioPlayer : NSObject

@property (nonatomic) AudioQueueRef queueRef;

// Play method

- (id)initWithAudio:(NSString *)filePath;

// Read Buffer

- (void)audioQueueOutputWithQueue:(AudioQueueRef)audioQueue queueBuffer:(AudioQueueBufferRef)audioQueueBufferRef;

- (UInt32)readPacketsIntoBuffer:(AudioQueueBufferRef)buffer;

// Callback function (Standard format)

static void BufferCallback(void *inUserData, AudioQueueRef inAQ, AudioQueueBufferRef buffer);

@end

這邊有一個重點

就是Callback的function部分, 依據Apple Official開放的官方文件

是不能做任何修改的, 我們能修改的只有命名

但是需要輸入的變數格式都是固定的, 這點請特別注意!!

接下來是我們要進去.m檔內實作裡面每一個Method囉!

#import "SHAudioPlayer.h"

// 3 is recommended by Apple

#define NUM_BUFFERS 3

static UInt32 gBufferSizeBytes = 0x10000; // It must be pow(2,x)

@interface SHAudioPlayer()

{

    // File ID

AudioFileID audioFileID;

    // Description

AudioStreamBasicDescription dataFormat;

    // Buffer Queue

AudioQueueRef                   queueRef;

SInt64                          packetIndex;

UInt32                          numPacketsToRead;

UInt32                          bufferByteSize;

AudioStreamPacketDescription    *packetsDescs;

AudioQueueBufferRef             buffers[NUM_BUFFERS];

}

@end

@implementation SHAudioPlayer

@synthesize queueRef;

// Init Player

- (id)initWithAudio:(NSString *)filePath

{

self = [super init];

if (self) {

UInt32   size;

UInt32   maxPacketSize;

char     *cookie;

OSStatus status;

        // Open File

status = AudioFileOpenURL((CFURLRef)[NSURL fileURLWithPath:filePath], kAudioFileReadPermission, 0, &audioFileID);

if (status != noErr) {

NSLog(@"Error encounter when opening file %@", filePath);

return nil;

}

        // Read audio data format

size = sizeof(dataFormat);

AudioFileGetProperty(audioFileID, kAudioFilePropertyDataFormat, &size, &dataFormat);

        // Create music queue for playing

AudioQueueNewOutput(&dataFormat, BufferCallback, self, nil, nil, 0, &queueRef);

        // Calculate number of packets in unit time

if (dataFormat.mBytesPerPacket == 0 || dataFormat.mFramesPerPacket == 0) {

size = sizeof(maxPacketSize);

AudioFileGetProperty(audioFileID, kAudioFilePropertyPacketSizeUpperBound, &size, &maxPacketSize);

if (maxPacketSize > gBufferSizeBytes) {

maxPacketSize = gBufferSizeBytes;

}

            // Calculation

numPacketsToRead = gBufferSizeBytes / maxPacketSize;

            // Allocate size

packetsDescs = malloc(sizeof(AudioStreamPacketDescription) * numPacketsToRead);

} else {

numPacketsToRead = gBufferSizeBytes / dataFormat.mBytesPerPacket;

packetsDescs = nil;

}

        // Set up Magic cookie

AudioFileGetProperty(audioFileID, kAudioFilePropertyMagicCookieData, &size, nil);

if (size > 0) {

cookie = malloc(sizeof(char) * size);

AudioFileGetProperty(audioFileID, kAudioFilePropertyMagicCookieData, &size, cookie);

AudioQueueSetProperty(queueRef, kAudioQueueProperty_MagicCookie, cookie, size);

}

        // Create and Allocate buffer size

packetIndex = 0;

for (int i = 0; i < NUM_BUFFERS; i++) {

AudioQueueAllocateBuffer(queueRef, gBufferSizeBytes, &buffers[i]);

            // Read Packet data

if ([self readPacketsIntoBuffer:buffers[i]] == 1) {

break;

}

}

Float32 gain = 1.0;

        // Set up volume

AudioQueueSetParameter(queueRef, kAudioQueueParam_Volume, gain);

        // Start queue, and then OS will handle the callback

AudioQueueStart(queueRef, nil);

}

    return self;

}

- (void)dealloc

{

[super dealloc];

}

// Callback Implement

static void BufferCallback(void *inUserData, AudioQueueRef inAQ, AudioQueueBufferRef buffer)

{

SHAudioPlayer *player = (SHAudioPlayer *)inUserData;

[player audioQueueOutputWithQueue:inAQ queueBuffer:buffer];

}

// Read Buffer

- (void)audioQueueOutputWithQueue:(AudioQueueRef)audioQueue queueBuffer:(AudioQueueBufferRef)audioQueueBufferRef

{

OSStatus status;

    // Read packet data

UInt32 numBytes;

UInt32 numPackets = numPacketsToRead;

status = AudioFileReadPackets(audioFileID,

NO,

&numBytes,

packetsDescs,

packetIndex,

&numPackets,

audioQueueBufferRef->mAudioData);

    // When succeed to read

if (numPackets > 0) {

        // Allocate buffer size as same as reading music buffer size

audioQueueBufferRef->mAudioDataByteSize = numBytes;

        // Finish allocating queue

status = AudioQueueEnqueueBuffer(audioQueue, audioQueueBufferRef, numPackets, packetsDescs);

        // Move Start index

packetIndex += numPackets;

}

}

- (UInt32)readPacketsIntoBuffer:(AudioQueueBufferRef)buffer

{

    // Read data and save it to buffer

UInt32 numBytes;

UInt32 numPackets = numPacketsToRead;

AudioFileReadPackets(audioFileID,

NO,

&numBytes,

packetsDescs,

packetIndex,

&numPackets,

buffer->mAudioData);

if (numPackets > 0) {

buffer->mAudioDataByteSize = numBytes;

AudioQueueEnqueueBuffer(queueRef, buffer, (packetsDescs ? numPackets : 0), packetsDescs);

packetIndex += numPackets;

} else {

return 1; // Which means we do not accept any packet

}

return 0; // Normal resign

}

@end

不過這張圖上並沒有標示出來我們實際是如何把buffer放到queue裡面的

這部分就請大家注意在AudioQueueNewOutput到AudioQueueStart間

我們用for 跑了3次, 是因為Apple建議將Buffer分成三份為最佳值

這邊裡面會執行一段method:

readPacketsIntoBuffer:(AudioQueueBufferRef)buffer 會把buffer 塞到queue內!

關於這個部分大家可以看到Bear下面畫的這張圖噢!

終於來到最後了!!當開始執行AudioQueueStart 後

一開始會先由這三段buffer播放

然後iPhone的OS會一直回傳Callback 給我們定義的function :BufferCallback

(Bear小提醒, 這邊Callback能改動的只有名稱喔! 輸入的變數順序跟型別都是固定不能改的!)

這邊我們做法就是當讀到Callback時

就繼續再把packet塞到下一個buffer內

這樣就能一直播放到歌曲結束囉!!!

如果要使用這個class的話, 只要在你的view controller中執行這一段:

SHAudioPlayer  *audioPlayer = [[SHAudioPlayer alloc] initWithAudio:@"/Users/shouian/Desktop/AppDev/AppCodeDevelop/SHAudioPlayer/brave.mp3"];

路徑可以隨意指定!
