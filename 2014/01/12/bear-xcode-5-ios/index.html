[入門] Bear實驗室: 如何用 Xcode 5 寫第一支iOS程式? 

Jan 12, 2014

鑑於前陣子 Bear 好友反應熱烈

決定從今天開始一系列完整的 iOS 有關教學

(其它相關實驗室文章也會持續推進!)

本篇教學給入門者學習如何第一次用 Xcode 5 寫iOS完全不需要基礎!

好奇學習的可以進來參考看看!

注意: 本篇教學假設你已經有一台 Mac 開發環境，如為 Windows 用戶須先建立環境首先請打開你的 Mac App Store

搜尋 Xcode 就能發現上圖

點選 Xcode 以後，就會進到本篇的第一張圖

點選安裝後就可以靜靜地一直按 “下一步” 了….

完成安裝以後並沒有這麼簡單

考慮到未來的使用，你需要下載以下工具：

Xcode command line tool :  (目前是5.0.1 GM版本)

1. [官方載點] (需要Apple 開發者帳號) 2. Mega載點

或是按下鍵盤上的 command + , 叫出 Preferences  視窗

切換到 Downloads 頁面，如下圖所示

按下 Command Line Tools 右邊的箭頭就會開始下載安裝

Command Line tool 這東西有什麼用，Bear 先在這邊不提

之後我們再介紹! 總之先裝了就對了! Xcode 裝好以後，會看到下圖

接下來點下去 XCode 就對囉!!

會跳出以下畫面

點下去位置以後會出來以下畫面

一開始要先選擇左上排的 Application

然後選擇上圖中，右上角的 Single View ApplicationXcode 5 在應用程式部分有支援8種樣板

但是 Bear 屬於老派人士依然支持用 Single View Controller 來制作

至於其它樣板的差異就必須留之後再討論了

依次介紹每個項目:Product Name: 專案名稱, 這邊 Bear 以著名的 HelloWorld 命名了

Organization Name: 公司組織名稱，這邊隨意取就好 Company Identifier:  這是識別字，要看底下的 Bundle Identifier Bundle Identifier: 這個是用來區分你的 App 跟其他 App 的不同, 一隻手機內只能有一個 Bundle Identifier (想像一下, 你的 iphone 內可以有 10 個名叫做 “LINE” 的App，他們可以內容完全一模一樣, 只要 Bundle Identifier 不同就好) Class Prefix: 這先不用理他 Device: 你要發佈的平台，可以是 iPad，iPhone 或是兩者都支援

注意:

Xcode 4.6 也就是一般的 iOS 6 版本以前底下還會有一個支援記憶體操作的選項

可以選擇 Automatic Reference Counting (自動管理記憶體 ARC) 或是

Manual Reference Counting (手動管理記憶體 MRC)

Xcode 5 以後預設都是用 ARC，要切換的話要到程式內部才能設定

創建完檔案以後，就依照下圖動作吧!

如果想要 fashion 一點

只要同時按下 command + r 按鈕就可以執行程式囉!
